k3s DevOps 闭环 ✅：CI/CD、滚动发布与回滚
我完成了一个端到端 DevOps 上线闭环：构建镜像并在 k3s 中部署 Deployment + Service(NodePort)，通过 Caddy 反向代理对外。 核心亮点是基于 GitHub Actions 与 Self-hosted Runner 实现了 Push-to-Deploy 自动化流水线，所有的静态资源上线与状态快照抓取均脱离手动操作，全自动完成。

我负责的内容
从“手动部署”跨越到“全自动化流水线”。
架构交付与 CI/CD
  • 持续集成/部署 (CI/CD)：接入 GitHub Actions，在服务器侧配置 Self-hosted Runner，实现代码提交后免密自动拉取、同步网页并执行底层脚本。
  • 集群编排：在 k3s 落地 Deployment + Service(NodePort)，保障副本数与滚动更新策略生效。
  • 公网暴露：Caddy 监听 80，/ 为演示页，/demo 反代到 127.0.0.1:30080。
运维与可用性保障
  • 可观测性构建：编写 Shell 脚本自动化抓取 Deployment / Pods / 发布记录，并通过 CI 流水线实时推送到前端页面展示。
  • 安全加固：UFW 防火墙严格收口，封闭 NodePort 直连；服务器禁用 Root 远程访问，启用专用普通用户权限隔离。
Push-to-Deploy 零人工干预 过程全记录
DevOps 与发布能力（我能做到什么）
  • 自动化流水线：Push 代码瞬间触发 GitHub Actions，云服务器秒级响应，自动完成站点更新。
  • 无缝滚动更新：执行镜像升级时，k3s 按预设 maxSurge 策略逐步替换 Pod，保障对外服务不断联。
  • 一键快速回滚：若遇故障，可迅速回滚至上一可用镜像版本（Rollback），止损迅速。
  • 发布留痕追踪：强制标注 change-cause，随时可通过 Rollout History 追溯历史变更。
